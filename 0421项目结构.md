# ROS2智能药品取用系统 - 项目模块分析

## 项目概述

本项目是一个**ROS2智能药品取用系统**，旨在实现智能化的药品管理、患者症状分析和用药建议。系统采用前后端分离架构，包含智能医疗助手代理(P1)和RESTful API后端(Backend)两大核心模块，通过ROS2与机器人系统集成。

**核心价值**:
- **数据驱动**: 从基础3种药品扩展到支持100+种药品管理
- **智能推荐**: 结合症状细节和药品信息，提供个性化推荐
- **管理便捷**: 支持批量导入、分类管理、库存监控
- **安全合规**: 集成禁忌症检查、相互作用验证等安全机制

## 主要模块概览

### 1. P1模块 - 智能医疗助手代理
**位置**: `agent_with_backend/P1/`

**核心组件**:
- **MedicalAgent** (`core/agent.py`): 医疗助手核心类，支持状态保存和恢复
- **症状提取服务** (`core/symptom_service.py`, `subagents/extractor.py`): 支持规则提取和LLM提取两种模式
- **工具系统** (`tools/`): 包含药品查询、过敏检查、剂量计算等工具
- **LLM客户端** (`llm/`): 支持Claude和OpenAI提供商
- **思考记录系统** (`thought_logging/`): 完整的LLM思考日志记录框架
- **工作流管理** (`core/workflows.py`): 管理医疗工作流状态

### 2. Backend模块 - 后端API服务
**位置**: `agent_with_backend/backend/`

**核心组件**:
- **API控制器**: 药品管理、订单处理、审批流程、健康检查
- **数据模型**: Drug（药品）、Order（订单）、Approval（审批）
- **ROS2集成** (`ros_integration/`): 与ROS2系统的桥梁
- **数据库管理**: SQLite数据库操作和过期药品清理

### 3. 其他模块
- **ROS集成层**: ROS-Todo和Unity模拟系统
- **技术文档**: 项目技术实施方案、架构设计文档

## 模块交互关系

### 系统架构图

```
用户输入 → P1 MedicalAgent → 症状提取 → 工具调用 → Backend API → ROS2系统
                     ↓
            思考记录系统 → 日志文件
```

### 主要交互流程:

1. **用户对话流程**:
   - 用户输入 → MedicalAgent.run() → 症状提取器提取结构化信息 → LLM分析 → 工具调用
   - 工具调用通过HTTP客户端与Backend API通信

2. **工具调用链**:
   - `query_drug` → `check_allergy` → `calc_dosage` → `generate_advice` → `submit_approval`
   - 每个工具调用对应Backend API端点

3. **数据流**:
   - P1通过 `PharmacyHTTPClient` (`utils/http_client.py`) 与Backend通信
   - Backend通过ROS2桥接器 (`ros_integration/`) 与机器人系统交互

### API通信接口
- **药品管理**: `GET /api/drugs`, `GET /api/drugs/{id}`
- **审批管理**: `POST /api/approvals`, `GET /api/approvals/{id}`
- **订单管理**: `POST /api/order`
- **健康检查**: `GET /api/health`

## 核心数据结构

### 1. 药品数据模型 (`backend/models/drug.py`)
```python
class Drug:
    drug_id: int        # 药品ID
    name: str           # 药品名称
    quantity: int       # 库存数量
    expiry_date: int    # 过期天数（0=已过期，负值=过期X天）
    shelf_x: int        # 货架X坐标
    shelf_y: int        # 货架Y坐标
    shelve_id: int      # 货架ID
```

### 2. 订单数据模型 (`backend/models/order.py`)
```python
class Order:
    task_id: int        # 任务ID
    status: str         # 状态（'pending', 'completed', 'failed'等）
    target_drug_id: int # 目标药品ID
    quantity: int       # 数量
    created_at: str     # 创建时间（ISO格式）
    drug_name: str      # 药品名称（可选，从其他表连接）
```

### 3. 审批数据模型 (`backend/models/approval.py`)
```python
class Approval:
    id: str             # 审批ID (格式: AP-YYYYMMDD-XXXXXX)
    patient_name: str   # 患者姓名
    advice: str         # 用药建议
    status: str         # 状态（'pending', 'approved', 'rejected'）
    created_at: str     # 创建时间（ISO格式）
    patient_age: int    # 患者年龄（可选）
    patient_weight: float  # 患者体重（可选）
    symptoms: str       # 症状描述（可选）
    drug_name: str      # 药品名称（可选）
    drug_type: str      # 药品类型（可选，'prescription', 'otc'）
    quantity: int       # 数量（默认1）
    doctor_id: str      # 医生ID（可选）
    reject_reason: str  # 拒绝原因（可选）
    approved_at: str    # 批准时间（可选，ISO格式）
```

### 4. 症状结构化数据 (`subagents/models.py`)
```python
class StructuredSymptoms:
    chief_complaint: str            # 主诉（主要症状）
    symptoms: List[str]             # 症状列表
    signs: Dict[str, Any]           # 体征（如体温、心率等）
    patient_info: PatientInfo       # 患者信息
    medical_history: Dict[str, Any] # 既往史

class PatientInfo:
    age: Optional[int]              # 年龄
    weight: Optional[float]         # 体重（kg）
    gender: Optional[str]           # 性别（'M', 'F', 'other'）
    allergies: Optional[List[str]]  # 过敏史列表
```

### 5. 工具定义 (`tools/registry.py`)
系统定义了以下工具供LLM调用：
1. `query_drug`: 根据症状或药品名称查询相关药物信息
2. `check_allergy`: 检查患者是否对某种药物过敏
3. `calc_dosage`: 根据患者年龄、体重、药品规格计算推荐剂量
4. `generate_advice`: 生成结构化的用药建议文本
5. `submit_approval`: 将AI生成的用药建议提交给医生审批
6. `fill_prescription`: 在医生审批通过后，将处方发送给药房系统进行配药

## 关键技术特征

### 1. 双模式症状提取
- **规则提取模式**: 快速但精度较低，使用正则表达式和关键词匹配
- **LLM提取模式**: 精度高但速度较慢，使用大语言模型进行语义理解
- **配置开关**: 通过 `ENABLE_LLM_SYMPTOM_EXTRACTION` 环境变量控制

### 2. 完整医疗工作流
```
收集患者信息 → 症状提取 → 药品查询 → 过敏检查 → 
剂量计算 → 建议生成 → 提交审批 → 等待审批结果
```

### 3. 状态持久化机制
- **会话状态保存**: MedicalAgent支持将对话状态保存到文件
- **工作流状态管理**: 跟踪每个患者的工作流进度
- **思考日志记录**: 完整的LLM思考过程记录和分析

### 4. 思考记录系统 (`thought_logging/`)
- **多格式输出**: 支持JSON和文本格式
- **多级别日志**: BASIC, STANDARD, DETAILED, DEBUG四个级别
- **自动清理**: 支持会话自动清理和保留期限配置

### 5. ROS2集成
- **消息适配器**: 将后端API消息转换为ROS2消息格式
- **健康监控**: 监控ROS2节点状态
- **任务发布器**: 将药品取用任务发布到ROS2系统

### 6. 前后端分离架构
- **前端(P1)**: 智能医疗助手，负责用户交互和逻辑处理
- **后端(Backend)**: RESTful API服务，负责数据存储和ROS2集成
- **通信方式**: HTTP客户端异步/同步调用

## 使用指南

### 启动后端服务
```bash
cd agent_with_backend/backend
python3 init_db.py        # 初始化数据库
python3 main.py           # 启动后端服务器（默认端口8001）
```

### 启动医疗助手
```bash
cd agent_with_backend/P1
python3 -m cli.patient_cli    # 启动命令行界面
```

### 环境配置
复制 `.thought_logging.env.example` 为 `.env` 并配置：
- `LLM_PROVIDER`: LLM提供商（claude/openai）
- `ANTHROPIC_API_KEY`: Claude API密钥
- `OPENAI_API_KEY`: OpenAI API密钥
- `PHARMACY_BASE_URL`: 后端API地址（默认 http://localhost:8001）
