# 测试计划执行快速参考指南

## 📋 一页纸总结

### 系统概述
**项目**：ROS2智能药品拣选系统  
**范围**：AI医疗助手 + 自动化药房后端 + ROS2集成  
**目标**：确保系统功能完整、性能稳定、质量达标  

### 关键测试阶段

| 阶段 | 周期 | 主要工作 | 完成指标 |
|------|------|--------|--------|
| **单元测试** | 第1周 | 测试各个API、工具函数、权限模块 | 覆盖率>85%, P1=0 |
| **集成测试** | 第2-3周 | 测试API调用链、数据库、流程整合 | API可用性>95%, P1=0 |
| **端到端测试** | 第4周 | 测试完整业务流程（患者→医生→系统） | 流程成功率>95%, P1=0 |
| **ROS2集成** | 第4周并行 | 测试机器人消息发布、错误降级 | 消息成功率>95% |

### 立即行动清单

```bash
# 1️⃣ 搭建环境（Day 1）
cd agent_with_backend
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt pytest pytest-cov
python database/scripts/init_db.py

# 2️⃣ 验证环境（Day 2）
pytest tests/ --co  # 查看所有可用测试
pytest --version && flask --version

# 3️⃣ 启动后端（Day 3）
python main.py &  # 启动Flask服务

# 4️⃣ 执行测试（Day 4+）
pytest tests/ -v --cov=api --cov-report=html
```

### 缺陷优先级定义

| 级别 | 定义示例 | 修复期限 |
|------|--------|--------|
| **P1 严重** | API完全挂、数据丢失、流程无法启动 | 立即修复 |
| **P2 主要** | 特定场景错误、响应慢、权限不对 | 发布前修复 |
| **P3 次要** | UI显示问题、文案不清楚 | 可延迟修复 |

### 报告生成

```bash
# 📊 生成HTML覆盖率报告
pytest tests/ --cov=agent_with_backend --cov-report=html
# 打开 htmlcov/index.html 查看

# 📋 生成测试报告
pytest tests/ --html=report.html --self-contained-html

# 📈 生成性能基准
pytest tests/ --benchmark-only --benchmark-json=bench.json

# 🔍 生成JUnit（用于CI）
pytest tests/ --junit-xml=results.xml
```

### 风险速查

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| LLM API不可用 | 中 | 高 | 使用Mock LLM |
| ROS2配置复杂 | 高 | 中 | 提前搭建 + 详细文档 |
| 数据库锁定 | 中 | 中 | 使用隔离测试实例 |
| 时间不足 | 中 | 中 | 优先P1/P2模块 |

### 常见命令速查

```bash
# 🏃 快速测试
pytest tests/ -v -x  # -x 首次失败停止

# 🎯 测试特定模块
pytest tests/test_drug_api.py -v

# 📊 显示详细覆盖率
pytest tests/ --cov=api --cov-report=term-missing

# 🔧 调试模式
pytest tests/test_drug_api.py -v -s --pdb  # -s 显示print, --pdb 断点

# ⚡ 并行执行（需要pytest-xdist）
pytest tests/ -n auto

# 🎪 性能测试
pytest tests/ --benchmark-only
```
