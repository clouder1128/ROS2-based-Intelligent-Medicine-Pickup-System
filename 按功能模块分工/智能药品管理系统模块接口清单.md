# 智能药品管理系统 - 模块接口清单

## 📋 概述

基于《智能药品管理系统 - 五人团队分工方案（按功能模块垂直分工）》文档，本文档定义了各模块间需要确定的API接口清单。按照方案要求，所有模块接口契约需要在第4周前正式确定。

## 🔗 模块依赖关系图

```
用户权限模块（模块3）
    ↓（提供认证API）
┌─────────────────────────────────────────┐
│          所有其他模块（1,2,4,5）         │
└─────────────────────────────────────────┘
    │                    │                    │
    ↓（提供药品数据）    ↓（调用药品数据）    ↓（调用药品数据）
药品管理模块（模块1）← 智能筛选模块（模块2）← 数据导入模块（模块4）
    │                    │                    │
    └────────────────────┴────────────────────┘
                         ↓（提供筛选结果）
                 系统界面模块（模块5）← 调用所有模块API
```

## 🧩 各模块接口清单

### 模块1：药品核心管理模块（提供药品数据API）

#### 需要提供的API接口

| 接口类别 | 接口端点 | HTTP方法 | 描述 | 请求参数 | 响应格式 |
|----------|----------|----------|------|----------|----------|
| **药品查询** | `/api/drugs` | GET | 获取药品列表（支持分页、筛选） | `page`, `limit`, `category`, `name` | `{success, drugs[], pagination}` |
| | `/api/drugs/{id}` | GET | 获取单个药品详情 | `id` | `{success, drug}` |
| **药品管理** | `/api/drugs` | POST | 创建新药品 | 药品完整信息 | `{success, drug_id}` |
| | `/api/drugs/{id}` | PUT | 更新药品信息 | `id`, 更新字段 | `{success, message}` |
| | `/api/drugs/{id}` | DELETE | 删除药品 | `id` | `{success, message}` |
| **库存管理** | `/api/inventory` | GET | 获取库存信息 | `drug_id`, `low_stock` | `{success, inventory[]}` |
| | `/api/inventory/{drug_id}/adjust` | POST | 调整库存 | `drug_id`, `quantity`, `type` | `{success, new_quantity}` |
| **分类管理** | `/api/categories` | GET | 获取药品分类 | - | `{success, categories[]}` |
| | `/api/categories` | POST | 创建新分类 | `name`, `description` | `{success, category_id}` |
| **搜索功能** | `/api/search/drugs` | GET | 综合搜索药品 | `keyword`, `filters` | `{success, drugs[], facets}` |

#### 需要调用的外部API接口
- **模块3（用户权限）**：需要调用认证API验证用户权限
- **无其他模块数据依赖**：是数据提供方

### 模块2：智能筛选与推荐模块（调用药品API，提供筛选API）

#### 需要提供的API接口

| 接口类别 | 接口端点 | HTTP方法 | 描述 | 请求参数 | 响应格式 |
|----------|----------|----------|------|----------|----------|
| **症状筛选** | `/api/screening/query` | POST | 根据症状筛选药品 | `symptoms`, `patient_info` | `{success, results[], confidence}` |
| | `/api/screening/batch` | POST | 批量症状筛选 | `queries[]` | `{success, batch_results[]}` |
| **历史管理** | `/api/screening/history` | GET | 获取筛选历史 | `user_id`, `limit` | `{success, history[]}` |
| | `/api/screening/history/{id}` | GET | 获取历史详情 | `id` | `{success, detail}` |
| **配置管理** | `/api/screening/config` | GET | 获取筛选配置 | - | `{success, config}` |
| | `/api/screening/config` | PUT | 更新筛选配置 | `config_updates` | `{success, message}` |
| **算法状态** | `/api/screening/status` | GET | 获取筛选服务状态 | - | `{success, status, metrics}` |

#### 需要调用的外部API接口
- **模块1（药品管理）**：
  - `GET /api/drugs` - 获取药品列表
  - `GET /api/drugs/{id}` - 获取药品详情
  - `GET /api/search/drugs` - 搜索药品
- **模块3（用户权限）**：需要调用认证API验证用户权限

### 模块3：用户权限与安全模块（提供认证API，被所有模块调用）

#### 需要提供的API接口

| 接口类别 | 接口端点 | HTTP方法 | 描述 | 请求参数 | 响应格式 |
|----------|----------|----------|------|----------|----------|
| **用户认证** | `/api/auth/login` | POST | 用户登录 | `username`, `password` | `{success, token, user_info}` |
| | `/api/auth/register` | POST | 用户注册 | `user_info` | `{success, user_id}` |
| | `/api/auth/logout` | POST | 用户登出 | `token` | `{success, message}` |
| | `/api/auth/refresh` | POST | 刷新令牌 | `refresh_token` | `{success, new_token}` |
| **用户管理** | `/api/users` | GET | 获取用户列表 | `role`, `status` | `{success, users[]}` |
| | `/api/users/{id}` | GET | 获取用户详情 | `id` | `{success, user}` |
| | `/api/users/{id}` | PUT | 更新用户信息 | `id`, `updates` | `{success, message}` |
| **权限管理** | `/api/roles` | GET | 获取角色列表 | - | `{success, roles[]}` |
| | `/api/permissions` | GET | 获取权限列表 | `role_id` | `{success, permissions[]}` |
| | `/api/users/{id}/permissions` | GET | 获取用户权限 | `id` | `{success, permissions[]}` |
| **审计日志** | `/api/audit/logs` | GET | 获取审计日志 | `user_id`, `action`, `date_range` | `{success, logs[]}` |
| | `/api/audit/stats` | GET | 获取审计统计 | `period` | `{success, stats}` |

#### 需要调用的外部API接口
- **无外部数据依赖**：是基础服务提供方

### 模块4：数据导入导出与批量处理模块（调用药品API，提供导入任务API）

#### 需要提供的API接口

| 接口类别 | 接口端点 | HTTP方法 | 描述 | 请求参数 | 响应格式 |
|----------|----------|----------|------|----------|----------|
| **数据导入** | `/api/import/templates` | GET | 获取导入模板 | `type` | `{success, templates[]}` |
| | `/api/import/validate` | POST | 验证导入数据 | `file`, `template_id` | `{success, validation_result}` |
| | `/api/import/execute` | POST | 执行数据导入 | `file`, `template_id`, `options` | `{success, job_id, stats}` |
| **导入任务** | `/api/import/jobs` | GET | 获取导入任务列表 | `status`, `user_id` | `{success, jobs[]}` |
| | `/api/import/jobs/{id}` | GET | 获取任务详情 | `id` | `{success, job_detail}` |
| | `/api/import/jobs/{id}/cancel` | POST | 取消导入任务 | `id` | `{success, message}` |
| **数据导出** | `/api/export/drugs` | POST | 导出药品数据 | `filters`, `format`, `fields` | `{success, download_url}` |
| | `/api/export/reports` | POST | 生成报表 | `report_type`, `parameters` | `{success, report_url}` |
| **批量操作** | `/api/batch/operations` | POST | 执行批量操作 | `operation`, `ids[]`, `parameters` | `{success, job_id}` |

#### 需要调用的外部API接口
- **模块1（药品管理）**：
  - `POST /api/drugs` - 创建药品（导入时调用）
  - `PUT /api/drugs/{id}` - 更新药品（导入时调用）
  - `POST /api/inventory/{drug_id}/adjust` - 调整库存
- **模块3（用户权限）**：需要调用认证API验证用户权限

### 模块5：系统界面与集成模块（调用所有模块API，提供统一前端）

#### 需要提供的API接口
*注：模块5主要负责前端界面，不提供后端API，但需要定义前端与后端的数据交互接口*

#### 前端路由接口（React Router）

| 页面路由 | 对应模块 | 需要调用的API | 页面功能 |

| `/screening` | 模块2 | `POST /api/screening/query` | 智能筛选页面 |

| `/users` | 模块3 | `GET /api/users` | 用户管理页面 |
| `/roles` | 模块3 | `GET /api/roles` | 角色管理页面 |
| `/import` | 模块4 | `GET /api/import/templates` | 数据导入页面 |
| `/export` | 模块4 | `POST /api/export/drugs` | 数据导出页面 |
| `/audit` | 模块3 | `GET /api/audit/logs` | 审计日志页面 |
| `/settings` | 系统设置 | 各模块配置API | 系统设置页面 |

#### 需要调用的外部API接口
- **所有模块API**：需要调用模块1-4的所有API接口
- **特别注意**：所有API调用都需要携带模块3提供的认证token

## 🎯 需要优先确定的核心接口

### 第一阶段（第1周确定）

#### 1. 公共数据模型接口
| 模型 | 字段定义 | 关联关系 |
|------|----------|----------|
| **药品模型** | `id`, `name`, `category`, `specification`, `price`, `stock`, `expiry_date`等30+字段 | 与分类、库存关联 |
| **用户模型** | `id`, `username`, `email`, `role`, `status`, `created_at` | 与角色、权限关联 |
| **分类模型** | `id`, `name`, `description`, `parent_id` | 层级结构 |
| **库存记录** | `id`, `drug_id`, `quantity`, `type`, `reason`, `operator` | 与药品、用户关联 |

#### 2. API基础规范接口
| 规范类型 | 具体约定 |
|----------|----------|
| **认证机制** | JWT令牌格式、刷新机制、有效期 |
| **错误处理** | 统一错误码、错误消息格式、HTTP状态码映射 |
| **分页规范** | `page`、`limit`参数、分页响应格式 |
| **时间格式** | ISO 8601标准时间格式 |
| **响应格式** | `{success: boolean, data: any, message?: string, error_code?: string}` |

### 第二阶段（第2周确定）

#### 1. 模块间关键接口
| 接口 | 提供模块 | 调用模块 | 接口定义 |
|------|----------|----------|----------|
| **药品查询接口** | 模块1 | 模块2,4,5 | `GET /api/drugs` 详细参数定义 |
| **用户认证接口** | 模块3 | 所有模块 | `POST /api/auth/login` 令牌格式 |
| **筛选查询接口** | 模块2 | 模块5 | `POST /api/screening/query` 请求响应格式 |

#### 2. 前端组件接口
| 组件类型 | 属性接口 | 事件接口 | 数据接口 |
|----------|----------|----------|----------|
| **药品表格** | `columns`, `data`, `pagination` | `onRowClick`, `onPageChange` | 调用`GET /api/drugs` |
| **筛选表单** | `symptoms`, `patient_info` | `onSubmit`, `onReset` | 调用`POST /api/screening/query` |
| **用户表单** | `user_data`, `validation_rules` | `onSubmit`, `onCancel` | 调用`POST /api/users` |

### 第三阶段（第4周前最终确定）

#### 所有模块完整接口契约
1. **模块1完整API文档**：所有药品管理相关接口
2. **模块2完整API文档**：所有智能筛选相关接口
3. **模块3完整API文档**：所有用户权限相关接口
4. **模块4完整API文档**：所有数据导入导出接口
5. **前端路由完整定义**：所有页面路由和API调用映射

## 🔧 接口测试和验证要求

### 接口测试清单
| 测试类型 | 测试内容 | 验证标准 |
|----------|----------|----------|
| **连通性测试** | 接口是否能正常访问 | HTTP状态码200，返回标准格式 |
| **参数验证** | 必填参数、参数类型、参数范围 | 正确参数通过，错误参数返回明确错误 |
| **权限验证** | 接口权限控制是否生效 | 无权限访问返回403，有权限正常访问 |
| **性能测试** | 接口响应时间、并发处理能力 | 响应时间<1s，支持并发请求 |
| **错误处理** | 异常情况下的错误返回 | 返回标准错误格式，包含错误码和消息 |

### 集成测试场景
1. **用户登录 → 药品查询**：验证认证和药品API集成
2. **症状筛选 → 药品详情**：验证筛选和药品详情API集成
3. **数据导入 → 库存更新**：验证导入和库存API集成
4. **完整业务流程**：用户登录 → 症状筛选 → 查看药品 → 数据导出

## 📄 接口文档管理

### 文档格式要求
1. **OpenAPI/Swagger规范**：所有接口必须提供OpenAPI规范文档
2. **接口变更日志**：记录接口变更历史，维护版本兼容性
3. **示例代码**：提供主要语言的调用示例（Python、JavaScript）
4. **错误码列表**：完整的错误码定义和解释

### 版本管理策略
1. **API版本前缀**：所有API使用版本前缀，如`/api/v1/drugs`
2. **向后兼容**：新增字段不破坏现有客户端，废弃字段标记`deprecated`
3. **版本迁移计划**：重大变更提供迁移路径和时间表
