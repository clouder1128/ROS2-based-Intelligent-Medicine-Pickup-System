# 后端详细实现规划

> **角色定位**：系统的「中枢神经」—— 处理用户请求、与数据库交互，作为 Web 世界（前端）和机器人世界（ROS 2）之间的「翻译官」。  
> 本文档整合开发计划与架构图，涵盖数据库、话题协议、实现任务与技术要点。

---

## 一、核心工作板块

### 1. 后端核心功能 (Flask 应用)

- **RESTful API**：接收前端 `id`、`num`，业务逻辑校验，坐标转换，任务下发
- **业务逻辑**：药品存在、未过期、库存充足
- **坐标映射**：药品编号 → `(shelve_id, shelf_x, shelf_y)` 或 `shelve_pos`
- **任务下发**：通过 ROS2 发布 `/task_request`

### 2. 状态反馈转换 (WebSocket / SSE)

- 后端作为 ROS2 **订阅者**，订阅 `/car_status`、`/shelve_status`
- 收到消息后通过 **WebSocket** 或 **SSE** 实时推送到前端
- **货柜状态也要反馈**（架构图强调）

### 3. 与 ROS2 的对接

- **发布**：`/task_request`（任务）、`/client_status`（用户确认取药）
- **订阅**：`/car_status`（小车状态）、`/shelve_status`（货柜状态）

## 二、数据库设计

### 药品表 (inventory)

| 字段 | 类型 | 说明 |
|------|------|------|
| drug_id | INTEGER | 主键 |
| name | TEXT | 药品名称 |
| quantity | INTEGER | 库存数量 |
| expiry_date | INTEGER | 倒计时（剩余天数），0 或负数表示已过期 |
| shelf_x | INTEGER | 货架 X 坐标 |
| shelf_y | INTEGER | 货架 Y 坐标 |
| shelve_id | INTEGER | 货架编号 |

### 任务/订单表 (order_log)

| 字段 | 类型 | 说明 |
|------|------|------|
| task_id | INTEGER | 主键，自增 |
| status | TEXT | pending / driving / arrived / completed / failed |
| target_drug_id | INTEGER | 目标药品 ID |
| quantity | INTEGER | 取药数量 |
| created_at | TIMESTAMP | 创建时间 |

### SQL Schema

```sql
CREATE TABLE inventory (
    drug_id INTEGER PRIMARY KEY,
    name TEXT NOT NULL,
    quantity INTEGER NOT NULL,
    expiry_date INTEGER NOT NULL,
    shelf_x INTEGER,
    shelf_y INTEGER,
    shelve_id INTEGER
);

CREATE TABLE order_log (
    task_id INTEGER PRIMARY KEY AUTOINCREMENT,
    status TEXT,
    target_drug_id INTEGER,
    quantity INTEGER,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (target_drug_id) REFERENCES inventory(drug_id)
);
```

## 三、整体数据流（取药流程）

```
用户确认取药 (id, num)
        │
        ▼
   Flask 后端 ──► 查库定位 (shelve_id, shelve_pos)
        │
        ├─► 发布 /task_request (type, id, shelve_id, shelve_pos, num)
        │
        ▼
   算法节点 ──► 控制小车前往货柜
        │
        ├─► 发布 car_status (行驶中) ──► 后端订阅 ──► WebSocket ──► 前端
        ├─► 发布 car_arrive (到达货柜) ──► 货柜放药
        ├─► 发布 shelve_status (已放) ──► 后端订阅 ──► WebSocket ──► 前端
        ├─► 发布 car_status (到达取药点) ──► 后端订阅 ──► WebSocket ──► 前端
        │
        ▼
   用户点击「已取药」
        │
        ▼
   Flask 后端 ──► 发布 client_status (cid, 已取=T) ──► 算法/小车
        │
        └─► 小车返回起点
```

## 四、话题与消息格式（与算法对齐）

### 4.1 后端 → 算法（发布）

| 话题 | 方向 | 类型 | 说明 |
|------|------|------|------|
| **/task_request** | 后端→算法 | std_msgs/String (JSON) | 下发取药/清理任务 |
| **/client_status** | 后端→算法 | std_msgs/String (JSON) | 用户确认取药后，通知小车返回 |

**/task_request 消息格式：**
```json
{
  "type": 1,
  "id": 1,
  "shelve_id": 1,
  "shelve_pos": [1, 1],
  "num": 2,
  "task_id": 1
}
```
- `type`: 0=清理过期药, 1=取药
- `shelve_pos`: 货架位置，可用 `[shelf_x, shelf_y]` 或格子编号

**/client_status 消息格式：**
```json
{
  "cid": 1,
  "已取": true
}
```

### 4.2 算法/小车/货柜 → 后端（订阅）

| 话题 | 方向 | 说明 |
|------|------|------|
| **/car_status** | 算法→后端 | 小车状态：行驶中、到达货柜、到达取药点 |
| **/car_arrive** | 小车→货柜（经算法转发） | 到达货柜，触发货柜放药 |
| **/shelve_status** | 货柜→后端 | 货柜状态：已放药等 |

**/car_status 消息格式：**
```json
{
  "cid": 1,
  "行驶中": true,
  "到达柜": 0,
  "到达药台": false,
  "当前目标柜": 1
}
```

**/shelve_status 消息格式：**
```json
{
  "sid": 1,
  "cid": 1,
  "status": "已放"
}
```

## 五、时序与状态映射（取药）

| 步骤 | 事件 | 话题/动作 | 后端处理 | order_log.status |
|------|------|-----------|----------|------------------|
| 1 | 小车开始行驶 | car_status (行驶中=T) | 订阅 → WebSocket 推前端 | pending → driving |
| 2 | 到达货柜 | car_arrive (sid, cid) | 可选记录 | driving |
| 3 | 货柜放药 | shelve_status (已放) | 订阅 → WebSocket 推前端 | driving |
| 4 | 继续行驶 | car_status (行驶中=T) | 推前端 | driving |
| 5 | 到达取药点 | car_status (到达药台=T) | 推前端，显示「已取药」按钮 | arrived |
| 6 | 用户确认取药 | 前端 POST /api/confirm_taken | 发布 client_status | arrived → completed |
| 7 | 小车返回起点 | car_status | 推前端 | completed |

## 八、技术实现要点

### rclpy 与 Flask 集成

- ROS2 须在**单独线程**中 spin，避免阻塞 Flask
- 发布者、订阅者均在同一 ROS2 节点内

```python
def ros2_thread():
    rclpy.init()
    node = BackendROS2Node()  # 包含 publisher + subscriber
    executor = SingleThreadedExecutor()
    executor.add_node(node)
    while rclpy.ok():
        executor.spin_once(timeout_sec=0.1)

t = threading.Thread(target=ros2_thread, daemon=True)
t.start()
```

### 实时反馈（WebSocket）

- 使用 **Flask-SocketIO**
- 在 ROS2 订阅回调中调用 `socketio.emit()` 推送到前端
