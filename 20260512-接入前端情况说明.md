# 2026-05-12 MedicalAgent 接入前端情况说明

## 启动方式

```bash
cd agent_with_backend
./quick_start.sh
```

启动后终端输出会显示前端和后端地址：
```
前端地址: http://localhost:8080
后端地址: http://localhost:8001/api/health
```

演示账号：
- 患者: patient1 / 123456
- 医生: doctor1 / 123456
- 管理员: admin1 / 123456

浏览器打开 `http://localhost:8080/web/` 进入患者端。按 Ctrl+C 停止所有服务。注意仅仅关闭网页，终端不会停止运行。
```

## 本次改动

### 1. AI 医疗咨询（`POST /api/consultation`）

- 暴露 MedicalAgent 为 REST 接口，LLM 驱动：症状提取 → 药品查询 → 过敏检查 → 剂量计算 → 生成建议 → 提交审批
- 支持多轮对话（同一 patient_id 保持会话上下文）
- 响应包含 `{ reply, approval_id, drug_name, quantity }` 结构化数据

### 2. 取药确认闭环

| 端点 | 功能 |
|------|------|
| `POST /api/orders/<task_id>/complete` | 患者确认取药，标记订单完成 |
| `POST /api/approvals/<id>/complete` | 标记审批单完成 |
| `POST /api/ros/return-to-queue` | 通知 ROS2 小车归队 |
| `POST /api/approvals/<id>/complete` | 同上（已在 approval_controller 中实现） |

### 3. 前后端一致性问题修复

- **drug_name 未知** — MedicalAgent 从 `submit_approval` 工具参数中提取 `drug_name`，API 返回结构化数据，前端不再依赖正则解析 LLM 回复
- **审批状态 403** — `request.auth_user` 不含 `display_name`，改为从 `auth_users` 表直接查询后注入 Agent，覆写 `submit_approval` 的 `patient_name`
- **启动时校验** — 页面加载时校验 token 有效性，清理 localStorage 中后端已不存在的审批记录（患者端 activeApprovals、医生端 approvalHistory）

### 4. 前端优化

- LLM 回复去除 markdown 符号
- 审批提交后结构化摘要（药品、剂量、用量），末尾声明 AI 生成、等待医生审批
- 取药状态跟踪显示药品剂量（布洛芬 x1）
- 模拟进度步骤（分析症状 → 查询药品 → 过敏检查 → 剂量计算 → 生成建议 → 提交审批）
