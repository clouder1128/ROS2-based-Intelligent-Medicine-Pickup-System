## 🏗️ 架构设计亮点

### 1. 分层架构

```
┌─ API路由层 (routes)
│  └─ 请求处理、参数验证、响应格式化
├─ 业务逻辑层 (services)
│  ├─ SymptomService: 症状处理
│  ├─ ScreeningService: 筛选算法
│  ├─ ConfigService: 配置管理
│  └─ HistoryService: 历史管理
├─ 数据模型层 (models)
│  ├─ Symptom: 症状模型
│  ├─ ScreeningHistory: 历史模型
│  └─ ScreeningConfig: 配置模型
└─ 数据存储层 (Database)
   └─ SQLAlchemy ORM + SQLite/MySQL
```

**优势**：
- 清晰的职责划分
- 易于测试和维护
- 可独立扩展各层

### 2. 筛选算法设计

**三层筛选机制**：
1. 症状层：精确匹配和模糊匹配
2. 条件层：价格、分类、效能过滤
3. 患者层：年龄、过敏信息考虑

**置信度计算**：
- 多因素加权平均
- 科学化的排序算法
- 支持自定义权重配置

### 3. 内存优化

- 症状库内存缓存
- 配置缓存机制
- 批量操作支持
- 分页查询缩减内存占用

### 4. 可扩展性

- 配置驱动的参数调整
- 支持自定义筛选算法
- 易于集成ML模型
- 支持多种数据库后端
