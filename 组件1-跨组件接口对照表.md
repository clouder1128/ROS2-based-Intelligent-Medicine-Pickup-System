### 1.2 统一响应格式

#### 组件1 `response.py` 实际输出格式

```json
{
  "success": true,
  "data":   { ... },
  "pagination": {
    "page": 1,   "limit": 20,  "total": 116,
    "pages": 6,  "has_next": true,  "has_prev": false
  },
  "error": null
}
```

#### 错误响应格式

```json
{
  "success": false,
  "data": null,
  "pagination": null,
  "error": { "code": "DRUG_NOT_FOUND", "message": "未找到 ID 为 999 的药品" }
}
```

#### 与各文档约定的对比

| 项目 | 接口清单约定 | 组件2文档约定 | 组件1实际 | 状态 |
|---|---|---|---|---|
| 成功响应顶层字段 | success / data / message? / pagination? | success / data / message / pagination | success / data / **error** / pagination | ⚠️ 用 `error` 代替 `message` |
| 错误字段结构 | `{error_code, message}` 扁平 | — | `{error: {code, message}}` 嵌套 | ⚠️ 嵌套结构 |
| 分页字段 | page/limit/total/pages/has_next/has_prev | 相同 | **完全一致** | ✅ |
| HTTP 状态码 | 200/201/400/401/403/404/500 | — | 200/201/400/401/403/404/500 | ✅ |

> ⚠️ **注意**：错误字段组件1用 `error: {code, message}` 嵌套，约定文档写的是 `message` 平铺。`drug_controller.py` 及各下单类接口普遍使用扁平格式：`{"success": false, "ok": false, "error": "<字符串>", "code": "<码>"}`，未调用 `response.py`。**组件2新写的 controller（category_controller.py 等）建议统一使用 `response.py`**；药品相关路由可与现有 `drug_controller._error` 保持一致，便于前端统一分支。

#### 软删除 `is_deleted`（实现口径）

以下路径**均只针对未删除药品**（条件为 `COALESCE(is_deleted, 0) = 0`，与列表过滤一致）：

| 能力 | 实现位置 | 说明 |
|---|---|---|
| 药品列表 / 症状筛选 | `api/drug_controller.py` `list_drugs` | SQL 或内存过滤均含 `is_deleted = 0` |
| 库存统计 | `api/drug_controller.py` `drug_stats` | `COUNT` / `SUM(quantity)` 仅统计未删除行 |
| 按 ID 取药校验 | `common/utils/drug_helpers.py` `query_drug` | 已删除则返回 `None`，下单/取药视为不存在 |
| 按名称找 ID | `common/utils/drug_helpers.py` `find_drug_id_by_name` | 精确/归一化/模糊匹配均排除已删除 |
| `validate_and_get_drug` | 同上（内部调用 `query_drug`） | 与 `/api/order`、`/api/pickup`、`/api/dispense` 一致 |

已软删除的药品：**列表不可见、统计不计入、不可通过订单接口扣库存**。

### 1.3 数据验证工具（组件1提供，供组件2使用）

| 函数 | 用途 | 调用方式 |
|---|---|---|
| `validate_drug(data, is_update=False)` | **唯一规则源**：与 `api/drug_controller` 内 `POST` / `PUT /api/drugs` **实现已耦合**（先校验再入库） | `from common.utils import validate_drug` |
| `validate_category(data, is_update=False)` | POST /api/categories / PUT 更新 | `from common.utils import validate_category` |
| `validate_inventory_transaction(data)` | POST /api/drugs/{id}/adjust 库存变更 | `from common.utils import validate_inventory_transaction` |
| `validate_pagination_params(args)` | 任意接口的 page/limit 参数预校验 | `from common.utils import validate_pagination_params` |

**与《五人团队分工方案（修订版）》组件1「编写完整的数据验证规则」对齐要点**：

| 项目 | 约定 |
|---|---|
| 创建必填 | `name`, `quantity`, `expiry_date`, `shelf_x`, `shelf_y`, `shelve_id` |
| `name` | 非空字符串，最大 **500** 字符（与 DB 写入截断一致） |
| `category` | 最大 **200** 字符；可为非字符串，按 `str(...).strip()` 再计长（与 controller 一致） |
| 大量 TEXT 扩展字段 | 单字段最大 **2000** 字符（与 `create_drug` 中 `optional_defaults` 路径一致） |
| `strength` | 最大 **100**；非字符串会先 `str` 再计长（与 `_strength_coerce` 一致） |
| `drug_interactions` / `age_restrictions` | 字符串或 JSON 可序列化对象，序列化后最大 **65535** |
| `indications` | 可选；若提供则为字符串数组，每项去除空白后最大 **500**（与 `_replace_indications` 一致） |
| 价格字段 | `retail_price` / `cost_price` / `purchase_price` 须为 **有效非负数字**，非法字符串 **400**（不再静默置 0） |

**校验失败时 HTTP 响应**（由 `drug_controller` 返回）：`400`，`code: "VALIDATION_ERROR"`，`error` 中含字段字典字符串。

**长度常量**（供 OpenAPI / 组件5 类型定义）：`from common.utils import DRUG_NAME_MAX_LEN, DRUG_CATEGORY_MAX_LEN, …`

**返回格式统一**：`(True, {})` 通过 / `(False, {"field": "错误说明"})` 失败

**使用示例**：
```python
from common.utils import validate_drug, bad_request_response

ok, errors = validate_drug(request.get_json())
if not ok:
    return bad_request_response(f"参数错误: {errors}")
```
