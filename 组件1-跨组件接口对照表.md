## 零、后端 HTTP 路由全量清单（与 `agent_with_backend/main.py` 注册蓝图一致）

以下路径均相对于服务根 URL（默认与 `Config` 中 `HOST`/`PORT` 一致）。**完整 URL** = **`/api`** + 下表「路径」列（例如药品列表为 **`GET /api/drugs`**）。**鉴权**列说明：`公开` = 无需 `Authorization`；`Bearer + 权限` = 须 `Authorization: Bearer <access_token>`，且 JWT 载荷中的权限集合包含对应操作码（`require_permission`）；`Bearer` = 仅需有效登录用户（`require_auth`）。**组件列**为建议主消费方，前端（组件5）通常会调用多个域。

**更细的请求体、Query、成功 JSON 形状**见本章 **§零·二**。

## 零·二、接口契约明细（请求 / 响应 / 鉴权）

本节在 **§零** 路由索引基础上，按**当前 `agent_with_backend` 实现**写清：**完整 URL**、**鉴权与权限码**、**Query/Body**、**成功体要点**。未列出的字段以各 `*_controller.py` / `auth/blueprint.py` / `screening_routes.py` 为准。

### 通用约定

| 项 | 约定 |
|---|---|
| 服务根 URL | `http://{HOST}:{PORT}`，默认见 `common/config.py` 中 `Config.HOST`、`Config.PORT`（常见 `8001`） |
| 路径前缀 | 下文「路径」均省略根，**完整路径** = 根 + 路径（多数为 **`/api/...`**；筛选为 **`/api/screening/...`**） |
| 鉴权头 | 需登录的接口：`Authorization: Bearer <access_token>`（登录见 `POST /api/auth/login`） |
| JSON | `Content-Type: application/json` |
| 权限码 | 与 `auth/constants.py` 及 JWT 内 `user_info.permissions` 一致，下表用字符串形式（如 `read:drug`） |

### 药品与库存（`drug_controller`）

| 方法 | 路径 | 鉴权（权限码） | Query / Body | 成功响应要点 |
|:---:|:---|:---|:---|:---|
| GET | `/api/drugs` | `read:drug` | Query：`name`、`symptom`、`category`、`sort_by`、`order`(asc/desc)、`page`、`limit`（传 `page`/`limit` 才分页；否则全量） | `success`、`data`（药品数组）、`count`、`filters`、`pagination?`、`error`（`list_drugs` 为 **`jsonify` 手写**，与 `response.py` 均含 `success/data/error`） |
| GET | `/api/inventory` | `read:inventory` | 与列表类似，另支持 `threshold`、`expiring_window` 等（拣货视图） | `success`、`data`（收窄字段 + 布尔标记）、`count`、`filters`、`pagination?`、`error` |

| POST | `/api/drugs` | `create:drug` | Body：与 `validate_drug(..., is_update=False)` 一致（必填含 `name`、`quantity`、`expiry_date`、`shelf_x`、`shelf_y`、`shelve_id` 等） | **`created_response`**：201，`data: { drug_id }`，`message` |
| PUT | `/api/drugs/<drug_id>` | `update:drug` | Body：部分字段；规则见 `validate_drug(..., is_update=True)` | `success`、`data:null`、`message` |
| DELETE | `/api/drugs/<drug_id>` | `delete:drug` | — | 软删：`is_deleted=1`；`success`、`data:null`、`message` |
| POST | `/api/drugs/<drug_id>/adjust` | `update:inventory` | Body：规范形 `quantity_change` + `transaction_type`（`in/out/adjust/expire`）；或兼容 `quantity` / `delta`（见接口 docstring） | `success`、`data`（含 `drug_id`、`quantity`、`quantity_change`、`transaction_id`）、`message`；写 `inventory_transactions` |
| POST | `/api/drugs/batch-import` | `batch:drug` | Body：**非空 JSON 数组**，每项同 POST 单条创建 | **`created_response`**：`data: { drug_ids, count }` |
| GET | `/api/drugs/export` | `read:drug` | `format=json|csv` 或 `Accept`；分页参数见代码 | JSON：`success`、`data: { drugs, count }`；CSV 为文件下载 |
| GET | `/api/drugs/stats` | `read:inventory` | — | `success`、`data`：`total_drugs`、`total_quantity`、`expired_count`、`low_stock_count`、`expiring_soon_count`；统计走缓存 |

> **缓存（组件1 第3周）**：`list_drugs` / `drug_stats` / `get_drug` 读缓存；`create/update/delete/adjust/batch-import` 后 `invalidate_drug_writes`。见 **§一·五**。
