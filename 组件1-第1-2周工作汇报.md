```sql
CREATE TABLE IF NOT EXISTS categories (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    name        TEXT    NOT NULL UNIQUE,   -- 与 inventory.category 字段值保持一致
    description TEXT    DEFAULT '',        -- 分类描述（接口清单约定字段）
    parent_id   INTEGER DEFAULT NULL,      -- 父分类（NULL = 顶级）

_add_column_if_not_exists(conn, "categories", "description", "TEXT DEFAULT ''")
```

**Week2：inventory_transactions 表**

```sql
CREATE TABLE IF NOT EXISTS inventory_transactions (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    drug_id         INTEGER NOT NULL,
    change          INTEGER NOT NULL,            -- 正=入库，负=出库
    change_type     TEXT NOT NULL DEFAULT 'adjust',  -- in/out/adjust/expire
    before_quantity INTEGER DEFAULT 0,
    after_quantity  INTEGER DEFAULT 0,
    reason          TEXT DEFAULT '',
    operator        TEXT DEFAULT '',
    created_at      TEXT DEFAULT '',

);
-- 索引
CREATE INDEX idx_inv_tx_drug_id     ON inventory_transactions(drug_id);
CREATE INDEX idx_inv_tx_created_at  ON inventory_transactions(created_at);
CREATE INDEX idx_inv_tx_change_type ON inventory_transactions(change_type);
```

### 4. `common/utils/validation.py`

**角色**：为组件2的 API 端点提供统一数据验证，减少各 Controller 重复的参数校验代码。

**改动**：从仅含2个占位函数的骨架，扩展为完整的 4 个验证器。

#### 对外提供的函数

##### `validate_drug(data, is_update=False) → (bool, dict)`

供 `POST /api/drugs`（创建）和 `PUT /api/drugs/<id>`（更新）使用。

| 字段 | 规则 |
|---|---|
| `name` | 创建时必填；非空字符串；≤**500** 字符（与 `drug_controller` 一致） |
| `quantity` / `stock` | 非负整数 |
| `retail_price` / `cost_price` / `purchase_price` | 非负浮点数 |
| `min_stock_level` / `max_stock_level` | 非负整数；min 不得大于 max |
| `dosage_form` | 枚举（片剂/胶囊/口服液/注射液 等16种） |
| `unit` | 枚举（盒/瓶/支/片/粒/袋/管/贴/个/克/毫升） |
| `pregnancy_category` | 枚举（A/B/C/D/X/空） |
| `is_prescription` | 只允许 0/1/True/False/"0"/"1" |
| 长文本字段 | 各有最大长度限制（description≤2000，usage_dosage≤1000 等） |

更新模式（`is_update=True`）时，所有字段均为可选，仅验证传入的字段。

##### `validate_category(data, is_update=False) → (bool, dict)`

供 `POST /api/categories`（创建）和 `PUT /api/categories/<id>`（更新）使用。

| 字段 | 规则 |
|---|---|
| `name` | 创建时必填，非空，≤50字符 |
| `description` | 可选，≤500字符 |
| `sort_order` | 非负整数 |
| `parent_id` | 正整数或 null |

##### `validate_inventory_transaction(data) → (bool, dict)`

供 `POST /api/drugs/<id>/adjust`（库存调整）使用。

| 字段 | 规则 |
|---|---|
| `change` | **必填**，非零整数（正=入库，负=出库） |
| `change_type` | **必填**，枚举：`in` / `out` / `adjust` / `expire` |
| `reason` | 可选，≤500字符 |
| `operator` | 可选，≤100字符 |

##### `validate_pagination_params(args) → (bool, dict)`

供任意分页查询接口的参数预校验使用。

| 参数 | 规则 |
|---|---|
| `page` | 正整数 |
| `limit` | 1～100 之间的整数 |

**与 Controller 绑定（修订，2026-05-08）**：`api/drug_controller.py` 中 **`POST /api/drugs`、`PUT /api/drugs/<id>`** 在执行业务逻辑前 **统一调用 `validate_drug(...)`**；校验失败返回 `400`、`code: "VALIDATION_ERROR"`。组件2新增批量导入等写接口时，应 **import `validate_drug`** 或复用 `common.utils` 中 `DRUG_*_MAX_LEN` 常量，避免规则分叉。

**调用方式**：
```python
from common.utils import validate_drug, validate_category, \
    validate_inventory_transaction, validate_pagination_params

ok, errors = validate_drug(request.get_json())
if not ok:
    return bad_request_response(str(errors))
```

### 7. `api/drug_controller.py`

**角色**：药品 REST API，由组件1/组件2协同维护。

#### （1）阻断性 Bug 修复（第1-2周交付时）

**Bug 描述**：`POST /api/drugs`、`PUT /api/drugs/<id>`、`DELETE /api/drugs/<id>` 各有两份函数定义。Flask 在 `register_blueprint` 时会因端点名重复抛出 `AssertionError`，导致**服务器完全无法启动**（阻断性 Bug）。

**原因**：组件2在添加完整版实现（30+字段、软删除）时，未删除原先的简化版（7字段、硬删除）。

**修复**：删除重复的旧实现，保留完整版：
- `create_drug`：保留支持 30+ 字段的完整版
- `update_drug`：保留支持所有可更新字段的完整版
- `delete_drug`：保留软删除版（`UPDATE SET is_deleted=1`，符合 schema 设计）

#### （2）软删除与统计一致（与代码同步修订，2026-05-08）

- **`GET /api/drugs/stats`**：`total_drugs`、`total_quantity`、过期/低库存/临期等计数均带条件 `COALESCE(is_deleted, 0) = 0`，与 `GET /api/drugs` 列表口径一致。
- **错误响应**：`stats` 异常时改为调用 `_error(...)`，与同文件其他接口一致（`success`、`ok`、`error` 字符串、`code`）。

**已通知组件2**：hotfix 不改变成功响应字段含义；统计口径收紧仅影响已软删除数据。

## 四、对外接口说明（组件1提供给其他组件）

### 4.1 提供给组件2（药品管理 API）

#### categories 表接口

**JOIN 约定**（无需修改 inventory 表结构）：
```sql
-- 查询药品时关联分类
SELECT i.*, c.id AS category_id, c.sort_order
FROM inventory i
LEFT JOIN categories c ON c.name = i.category
WHERE i.is_deleted = 0;

-- 按分类统计药品数
SELECT c.id, c.name, COUNT(i.drug_id) AS drug_count
FROM categories c
LEFT JOIN inventory i ON i.category = c.name AND i.is_deleted = 0
GROUP BY c.id ORDER BY c.sort_order;
```

> 约定：`inventory.category`（TEXT）= `categories.name`（TEXT），字符串直接匹配，无需 `category_id` 外键。

#### inventory_transactions 表接口

组件2在调用库存调整后，应向此表写入一条记录：
```python
conn.execute("""
    INSERT INTO inventory_transactions
        (drug_id, change, change_type, before_quantity, after_quantity, reason, operator, created_at)
    VALUES (?, ?, ?, ?, ?, ?, ?, ?)
""", (drug_id, delta, change_type, before_qty, after_qty, reason, operator, utc_now))
```

`change_type` 枚举值：`'in'`（入库）、`'out'`（出库）、`'adjust'`（手动调整）、`'expire'`（过期报废）

#### 数据验证工具

```python
from common.utils import validate_drug, validate_category, validate_inventory_transaction

```

#### 统一响应格式

```python

    success_response,       # 200
```
