## 一、概述

### 1.1 系统简介
ROS2智能药品拣选系统是一个集成的医疗用药管理系统，包含：
1. **智能药房后端**：Flask服务（端口8001），提供药品管理、订单处理、审批API
2. **P1医疗助手**：AI Agent系统，提供症状分析、药品推荐、剂量计算
3. **ROS2集成**：自动化取药任务发布到仿真系统
4. **定时任务**：自动过期药品清扫和库存管理

### 1.2 架构概览
```
┌─────────────────┐     HTTP API     ┌─────────────────┐
│   P1 Medical    │◄────────────────►│   Smart Pharmacy │
│    Assistant    │   (端口8001)     │    Backend      │
│                 │                  │                 │
│  • LLM Agent    │                  │  • Flask API    │
│  • Medical Tools│                  │  • SQLite DB    │
│  • Workflows    │                  │  • ROS2 Bridge  │
│  • Memory       │                  │  • Timed Tasks  │
└─────────────────┘                  └────────┬────────┘
         │                                    │ ROS2
         │                                    │ /task_request
         └────────────────────────────────────┘
                     (Future: Simulation)
```

### 1.3 核心组件
- **后端服务**：`backend/app.py`（Flask应用）
- **数据库**：`backend/pharmacy.db`（SQLite）
- **AI助手**：`P1/core/agent.py`（MedicalAgent）
- **HTTP客户端**：`P1/utils/http_client.py`（PharmacyHTTPClient）
- **审批模块**：`backend/approval.py`（ApprovalManager）
- **测试套件**：`P1/tests/` + `backend/tests/`

## 三、API参考手册

### 3.1 基础信息
- **基础URL**: `http://localhost:8001`（开发环境）
- **认证**: 当前无认证，生产环境中请实现API密钥或JWT
- **响应格式**: 所有响应均为JSON格式。成功响应包含 `"success": true`，错误响应包含 `"error": true` 和 `"message"`
- **CORS**: 已配置支持跨域访问

### 3.2 健康检查
`GET /api/health`

**响应示例**:
```json
{
  "success": true,
  "backend": "pharmacy",
  "ros2_connected": false,
  "timestamp": "2026-04-07T10:30:00"
}
```

**用途**: 检查后端服务是否正常运行，ROS2连接状态

### 3.3 药品管理API

#### 获取所有药品
`GET /api/drugs`

**查询参数**:
- `name`（可选）：按药品名称筛选（部分匹配）

**响应示例**:
```json
{
  "success": true,
  "drugs": [
{
  "drug_id": 1,
  "name": "阿莫西林",
  "quantity": 50,
  "expiry_date": 365,
  "shelf_x": 1,
  "shelf_y": 1,
  "shelve_id": 1
}
  ],
  "count": 1
}
```

#### 获取单个药品
`GET /api/drugs/{id}`

**响应**: 同上，但使用 `"drug"` 键而非 `"drugs"`

### 3.4 审批管理API

#### 创建审批
`POST /api/approvals`

**请求体**:
```json
{
  "patient_name": "John Doe",
  "advice": "Take medication",
  "patient_age": 30,
  "patient_weight": 70.5,
  "symptoms": "Headache",
  "drug_name": "Ibuprofen",
  "drug_type": "NSAID"
}
```

**必填字段**: `patient_name`, `advice`

**响应**:
```json
{
  "success": true,
  "approval_id": "AP-20260407-ABCD1234",
  "message": "Approval created successfully",
  "created_at": "2026-04-07T10:30:00"
}
```

#### 获取审批详情
`GET /api/approvals/{id}`

**响应**:
```json
{
  "success": true,
  "id": "AP-20260407-ABCD1234",
  "patient_name": "John Doe",
  "status": "pending",
  "created_at": "2026-04-07T10:30:00",
  // ... 所有审批字段
}
```

#### 获取待处理审批
`GET /api/approvals/pending`

**查询参数**:
- `limit`（可选，默认 100）：最大返回数量

**响应**:
```json
{
  "success": true,
  "approvals": [...],
  "count": 5,
  "limit": 100
}
```

#### 批准审批
`POST /api/approvals/{id}/approve`

**请求体**:
```json
{
  "doctor_id": "dr_smith"
}
```

**响应**:
```json
{
  "success": true,
  "message": "Approval approved successfully",
  "approval_id": "AP-20260407-ABCD1234",
  "doctor_id": "dr_smith",
  "approved_at": "2026-04-07T10:35:00"
}
```

#### 拒绝审批
`POST /api/approvals/{id}/reject`

**请求体**:
```json
{
  "doctor_id": "dr_jones",
  "reason": "Patient has contraindication"
}
```

**响应**:
```json
{
  "success": true,
  "message": "Approval rejected successfully",
  "approval_id": "AP-20260407-ABCD1234",
  "doctor_id": "dr_jones",
  "reason": "Patient has contraindication",
  "rejected_at": "2026-04-07T10:35:00"
}
```

### 3.5 订单管理API

#### 创建订单（批量取药）
`POST /api/order`

**请求体**:
```json
[
  {"id": 1, "num": 2},
  {"id": 2, "num": 1}
]
```

**响应**:
```json
{
  "success": true,
  "task_ids": [1, 2],
  "message": "已下发 2 个取药任务，库存已扣减"
}
```

#### 获取订单历史
`GET /api/orders`

**响应**:
```json
{
  "success": true,
  "orders": [
{
  "task_id": 1,
  "status": "completed",
  "target_drug_id": 1,
  "quantity": 2,
  "created_at": "2026-04-07T10:00:00"
}
  ],
  "count": 1
}
```

### 3.6 错误响应参考

#### 400 错误请求
```json
{
  "error": true,
  "message": "Missing required field: patient_name",
  "code": "MISSING_FIELD"
}
```

#### 404 未找到
```json
{
  "error": true,
  "message": "Drug not found with ID: 999",
  "code": "DRUG_NOT_FOUND"
}
```

#### 500 内部服务器错误
```json
{
  "error": true,
  "message": "Database connection failed",
  "code": "DB_CONNECTION_ERROR"
}
```

## 四、集成开发指南

### 4.1 后端-P1集成架构
- **后端**: 运行在端口8001的Flask服务，提供药品管理、订单和审批的REST API
- **P1**: 医疗助手，通过HTTP客户端组件调用后端API
- **数据流**: P1工具 → HTTP API → 后端 → 数据库/ROS2
- **配置**: 通过`PHARMACY_BASE_URL`环境变量连接（默认`http://localhost:8001`）

## 五、故障排除与维护

### 5.1 问题分类索引

**按问题类型快速定位**:
1. **后端启动问题**：端口冲突、依赖缺失、模块导入错误
2. **连接问题**：P1无法连接后端、CORS错误、网络问题
3. **数据库问题**：数据库锁定、表不存在、数据不一致
4. **API问题**：审批API错误、订单API失败、错误响应处理
5. **性能问题**：响应缓慢、内存使用率高
6. **测试问题**：集成测试失败、模拟测试错误
7. **ROS2集成问题**：ROS2未安装、话题发布失败

### 5.2 后端问题

#### 5.2.1 启动问题
**问题**: `Address already in use` (端口被占用)

**解决方案**:
```bash
# 检查端口8001被什么占用
sudo lsof -i :8001

# 终止占用进程
kill -9 <PID>

# 或者使用不同端口
export PORT=8002
cd backend && python app.py
```

**问题**: `ModuleNotFoundError: No module named 'flask'`

**解决方案**:
```bash
# 安装依赖
cd backend
pip install -r requirements.txt

# 如果requirements.txt不存在或损坏
pip install flask flask_cors sqlite3
```

#### 5.2.2 其他启动错误
**问题**: `ImportError: No module named 'rclpy'`

**解决方案**:

```bash
#!/bin/bash
# monitor-pharmacy.sh

HEALTH_URL="http://localhost:8001/api/health"
LOG_FILE="/var/log/pharmacy-monitor.log"

check_health() {
response=$(curl -s -o /dev/null -w "%{http_code}" $HEALTH_URL)
if [ "$response" -eq 200 ]; then
    echo "$(date): ✅ 服务正常 (HTTP $response)" >> $LOG_FILE
    return 0
else
    echo "$(date): ❌ 服务异常 (HTTP $response)" >> $LOG_FILE
    return 1
fi
}

check_process() {
if pgrep -f "python app.py" > /dev/null; then
    echo "$(date): ✅ 进程运行中" >> $LOG_FILE
    return 0
else
    echo "$(date): ❌ 进程未运行" >> $LOG_FILE
    return 1
fi
}

# 执行检查
check_health
check_process
```
