# 智能药品管理系统技术实施方案

## 📋 项目概述

### **项目背景**
当前ROS2智能药品取用系统面临两个关键需求：
1. **药品数量不足，缺乏管理**：仅3种基础药品，缺乏完整CRUD管理功能
2. **药品推荐逻辑存在问题**：基于硬编码映射，缺乏医学合理性判断

### **整合目标**
将**药品管理API扩展**与**智能药品筛选系统**两个方案整合，构建完整的药品管理生态：
- **基础层**：扩展药品数据结构，支持100+种药品的动态管理
- **智能层**：实现基于LLM的智能药品筛选，提升推荐准确性
- **管理层**：提供完整的药品CRUD操作和批量导入功能
- **安全层**：集成权限控制，确保系统安全性

### **核心价值**
1. **数据驱动**：从3种药品扩展到100+种，支持完整药品知识库
2. **智能推荐**：结合症状细节和药品信息，提供个性化推荐
3. **管理便捷**：支持批量导入、分类管理、库存监控
4. **安全合规**：集成禁忌症检查、相互作用验证等安全机制

```sql
-- 用户表（权限管理）
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username TEXT UNIQUE NOT NULL,
    password_hash TEXT NOT NULL,
    full_name TEXT,
    email TEXT,
    role TEXT NOT NULL,  -- 'admin','doctor','pharmacist','viewer'
    department TEXT,
    is_active BOOLEAN DEFAULT 1,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    last_login_at TIMESTAMP
);

-- 操作审计表
CREATE TABLE audit_logs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER,
    action_type TEXT NOT NULL,  -- 'create_drug','update_drug','delete_drug','batch_import'
    resource_type TEXT NOT NULL, -- 'drug','order','approval'
    resource_id TEXT,
    details TEXT,  -- JSON格式的操作详情
    ip_address TEXT,
    user_agent TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);

-- 药品导入日志表
CREATE TABLE drug_import_logs (
    import_id TEXT PRIMARY KEY,
    filename TEXT,
    format TEXT,  -- 'csv','json'
    total_records INTEGER,
    successful_records INTEGER,
    failed_records INTEGER,
    imported_by INTEGER,
    started_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    completed_at TIMESTAMP,
    status TEXT,  -- 'pending','processing','completed','failed'
    error_log TEXT,
    FOREIGN KEY (imported_by) REFERENCES users(id)
);
```

## 🔧 API设计规范

### **1. 药品管理API** (`/api/drugs`)

| 端点 | 方法 | 功能 | 权限 |
|------|------|------|------|
| `/api/drugs` | GET | 获取药品列表（分页、筛选、排序） | 所有用户 |
| `/api/drugs` | POST | 创建新药品 | admin, pharmacist |
| `/api/drugs/<id>` | GET | 获取药品详情 | 所有用户 |
| `/api/drugs/<id>` | PUT | 更新药品信息 | admin, pharmacist |
| `/api/drugs/<id>` | DELETE | 软删除药品 | admin |
| `/api/drugs/batch-import` | POST | 批量导入药品 | admin |
| `/api/drugs/export` | GET | 导出药品数据 | admin, pharmacist |
| `/api/drugs/categories` | GET | 获取分类统计 | 所有用户 |
| `/api/drugs/low-stock` | GET | 获取低库存药品 | admin, pharmacist |
| `/api/drugs/expiring-soon` | GET | 获取即将过期药品 | admin, pharmacist |

### **2. 智能筛选API** (`/api/screening`)

| 端点 | 方法 | 功能 | 权限 |
|------|------|------|------|
| `/api/screening/query` | POST | 智能药品查询与筛选 | doctor, pharmacist |
| `/api/screening/history` | GET | 筛选历史记录 | 所有用户 |
| `/api/screening/config` | GET | 获取筛选配置 | admin |
| `/api/screening/config` | PUT | 更新筛选配置 | admin |

### **3. 权限认证API** (`/api/auth`)

| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/auth/login` | POST | 用户登录 |
| `/api/auth/logout` | POST | 用户登出 |
| `/api/auth/register` | POST | 用户注册（管理员） |
| `/api/auth/profile` | GET | 获取用户资料 |
| `/api/auth/profile` | PUT | 更新用户资料 |

### **4. 保持向后兼容的现有API**
- `/api/drugs` (GET) - 保持现有格式兼容
- `/api/order` (POST) - 药品下单
- `/api/approvals/*` - 审批流程
